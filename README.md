## üß≠ Key files & where to look (simple, beginner-friendly)

I‚Äôve kept the explanation short and focused on what matters most when you open an Android project.

- Java source code (app/src/main/java/)
    - What it is: The app‚Äôs behavior lives here. Look for Activity classes (screens), Adapters (list logic), and small helper/util classes.
    - Where to start: Find `MainActivity` ‚Äî that‚Äôs the app entry point. Open it to see which layout it loads and how user actions are handled.
    - Why : Java files show your logic, structure, and coding.

- Resource XML (app/src/main/res/)
    - What it is: All UI definitions and static resources.
    - Important folders:
        - `res/layout/` ‚Äî screen and component layouts (XML you can edit visually in Android Studio).
        - `res/values/` ‚Äî strings, colors, styles (keeps text and theme separate from code).
        - `res/drawable/` & `res/mipmap/` ‚Äî icons and images.
    - Why: XML shows how UI is built and separated from code. It‚Äôs the quickest way to change visuals without touching Java.

- AndroidManifest.xml
    - What it is: Declares app components (Activities) and permissions.
    - Quick use: Use it to find the launcher Activity or to check if special permissions are required.

That‚Äôs it ‚Äî focus on Java + XML + Manifest to understand each sample quickly.
